## Short description

Agent-scoped local retrieval for OpenClaw: explicit corpora, deny-by-default access, and a validated minimal demo.

## Medium description

`Agent-Scoped Local Retrieval` helps OpenClaw users build local retrieval that is scoped, explainable, and safer to maintain.

It focuses on a few things that matter in real systems:
- separate personal memory from workspace knowledge
- use explicit corpora instead of indexing everything
- restrict access per agent
- gate execution on prerequisite checks
- validate a minimal closed loop with runnable smoke tests

Good fit for multi-agent workspaces, local RAG, and retrieval systems that need clearer trust boundaries.

## Long description

`Agent-Scoped Local Retrieval` is a reusable retrieval architecture skill for OpenClaw.

What it gives you:
- explicit corpus boundaries
- deny-by-default agent access
- a stable search-wrapper pattern
- selective refresh guidance
- prerequisite checks before execution
- validation rules for honest maturity claims
- a fully validated minimal demo path

What it is not:
- not a hosted vector database
- not a turnkey enterprise product
- not “index everything” local RAG

Use it when you want retrieval that looks disciplined, not improvised.
